# 首页壳层与 HLTV 页面设计

- 日期：2026-03-19
- 项目：CS2DemoPlayer
- 状态：设计已确认，待用户审阅后进入实现计划

## 1. 背景

当前主界面仍然是一个偏工具化的单面板页面：

- 首页只有居中的 Demo 库卡片
- HLTV 最小抓取与下载原型已经存在，但还没有正式页面承载
- 回放页已经独立存在，不适合再塞回首页信息架构里

项目下一阶段的优先方向已经明确为：

1. 继续优化前端完整性
2. 接入 HLTV 数据
3. 后续再推进 md、tags、search

这次设计只处理前两项在首页层面的落地方式：把首页升级为现代化应用壳层，并为 HLTV 建立一个正式内容页。

## 2. 目标与非目标

### 目标

- 把首页改造成完整的双栏应用壳层
- 左侧提供可折叠导航，首批页面为 `Demo 库` 与 `HLTV`
- 默认进入 `Demo 库`
- `HLTV` 页面展示最近比赛，并提供下载 demo 的主链路
- `回放页` 继续保持独立视图，不并入首页标签页

### 非目标

- 本轮不重做回放页结构
- 本轮不设计 `Notes / Tags / Search` 的正式页面
- 本轮不把 HLTV 页面做成复杂仪表盘、赛事中心或数据分析页
- 本轮不引入超出当前需求的账号体系、收藏、分页筛选、个性化推荐

## 3. 方案比较

### 方案 A：最小改造

做法：

- 保留现有顶部工具栏
- 在 `home-view` 内部加左侧导航和右侧内容区
- `Demo 库` 复用现有列表
- `HLTV` 补一个基础列表页

优点：

- 改动最小
- 上线最快

缺点：

- 页面仍会保留明显“工具面板拼接感”
- 很难达到“现代化网页”的整体观感

### 方案 B：中度重构，推荐

做法：

- 主页升级为完整 app shell
- 左侧是可折叠导航
- 右侧是内容区，承载 `Demo 库` 与 `HLTV`
- 顶部保留全局工具，但职责收敛
- 回放页维持独立

优点：

- 结构清晰，现代感明显提升
- 能自然承接未来的 `Notes / Tags / Search`
- 与当前代码结构仍保持连续，不需要一次性推翻 renderer

缺点：

- 需要调整首页 DOM 结构与样式组织

### 方案 C：全面产品化首页

做法：

- 首页做成仪表盘，包含 hero、统计卡、Demo 预览、HLTV 预览

优点：

- 视觉完整度最高

缺点：

- 过度设计
- 与“默认直接进入 Demo 库”的用户意图冲突

### 选择

采用方案 B。

原因：

- 它同时满足“像现代化网页”和“默认先看 Demo 库”两个要求
- 它能为 HLTV 提供正式页面，同时不干扰现有回放主链路

## 4. 信息架构

### 4.1 顶层视图

应用保留两个顶层视图：

- 首页壳层视图
- 回放视图

首页壳层视图负责内容浏览、导入入口和 HLTV 获取；回放视图负责单个 demo 的加载与播放。

### 4.2 首页壳层

首页壳层采用两栏结构：

- 左栏：可折叠导航
- 右栏：内容区

首批导航项固定为：

- `Demo 库`
- `HLTV`

后续如需扩展 `Notes`，只作为新增导航项，不改变本轮结构。

### 4.3 默认落地页

应用打开首页后默认进入 `Demo 库`，不做独立仪表盘首页。

这个决策的目的，是让首页直接服务当前最高频链路：

- 查看本地 demo
- 进入回放
- 从 HLTV 下载后返回本地库继续处理

## 5. 页面设计

### 5.1 左侧导航

左侧导航负责全局切页，不承载页面内操作。

包含以下元素：

- 品牌区：应用标题或简短品牌文本
- 导航项：`Demo 库`、`HLTV`
- 可折叠按钮

行为规则：

- 展开时显示图标和文字
- 折叠时保留图标或紧凑标记
- 折叠状态在当前会话中保持
- 当前选中项有清晰高亮

### 5.2 全局顶部栏

顶部栏保留，但职责收敛为全局层级：

- 返回首页 / 回放切换相关控制
- 导入 demo
- 全局状态文本

页面专属操作不再堆在全局栏，而应放回各自页面头部。

### 5.3 Demo 库页

`Demo 库` 页是默认内容页。

布局：

- 页头
- 轻量状态摘要
- 主体列表区

页头内容：

- 左侧：页面标题和一句简短说明
- 右侧：`导入 demo`、`刷新`

状态摘要只保留轻量信息，例如：

- 本地 demo 总数
- 已解析数或缓存状态概览
- 最近更新时间

主体列表区不再使用窄居中的单盒子，而是铺满右侧内容区，作为内容工作台。

每个 demo 条目展示：

- 名称
- 地图
- 回合数 / 缓存状态
- 更新时间
- 解析状态

保留现有交互：

- 点击条目进入回放
- 当前选中条目高亮
- 右键菜单继续支持重命名与删除

空状态要求：

- 不是单行文本
- 要有清晰空状态卡片
- 提供 `导入 demo` 主按钮
- 明确说明也可以先到 `HLTV` 页面获取 demo

### 5.4 HLTV 页

`HLTV` 页第一版只负责一条最短链路：

- 获取最近比赛
- 展示最近比赛列表
- 下载可用 demo
- 让下载结果进入本地使用流程

布局：

- 页头
- 状态区
- 最近比赛列表

页头内容：

- 左侧：标题 `HLTV` 与一句页面说明
- 右侧：`获取最近比赛`

状态区负责展示：

- 当前抓取状态：空闲、加载中、成功、失败
- 最近一次抓取结果摘要
- 失败原因

最近比赛列表中，每项至少包含：

- 两队名称
- 赛事名称
- 比赛时间或“最近完成”文本
- demo 可用状态
- 主操作按钮：`下载 demo`

下载成功后的状态应转为：

- `已下载`
- 可提供“前往 Demo 库查看”或“在库中打开”的下一步入口

第一版不做以下能力：

- 赛事筛选
- 多条件搜索
- 分页
- 图片封面
- 复杂比赛详情展开

## 6. 数据与状态边界

### 6.1 首页壳层状态

首页壳层只管理以下前端状态：

- 当前选中的导航页
- 左栏折叠状态
- 页面级加载与空状态

它不直接承担回放状态管理。

### 6.2 Demo 库页状态

`Demo 库` 页继续消费现有 demo library 数据源。

这意味着本轮不重写 demo 数据模型，只调整呈现方式与页面层组织。

### 6.3 HLTV 页状态

`HLTV` 页第一版只消费已存在的 HLTV 最小原型结果。

需要的最小数据对象包括：

- 比赛标识
- 两队名称
- 赛事名称
- 比赛链接
- demo 下载状态
- 下载后的本地路径或关联状态

页面不负责解释 Playwright 或抓取细节，只消费主进程暴露的高层结果。

### 6.4 回放页边界

回放页保持独立。

进入方式仍然来自：

- `Demo 库` 页中选择本地 demo
- 后续 HLTV 下载完成后，回到本地库再进入

本轮不让 HLTV 页直接承担回放容器职责。

## 7. 组件拆分建议

为避免继续把首页逻辑堆进单一文件，建议首页改造时按下面边界拆分：

### `home shell`

职责：

- 管理左栏折叠
- 管理当前页面切换
- 组织页头与内容区容器

### `demo library page`

职责：

- 渲染 Demo 库页头、摘要、列表、空状态
- 复用现有 demo library 数据和进入回放逻辑

### `hltv page`

职责：

- 触发最近比赛获取
- 展示抓取状态
- 渲染最近比赛列表和下载动作

### `global toolbar`

职责：

- 放置全局级动作
- 不持有页面专属渲染逻辑

这些单元都应做到：

- 单一职责
- 边界清晰
- 可以在不读内部实现的情况下理解用途

## 8. 交互规则

### 8.1 导航切换

- 切换 `Demo 库` 与 `HLTV` 不应刷新整个应用
- 内容区做页面切换，保留统一壳层
- 当前页面状态应尽量保留，不做无意义重置

### 8.2 Demo 库交互

- `Demo 库` 是默认选中项
- 刷新只影响库数据，不重置整个壳层
- 进入回放时离开首页壳层，切到独立回放视图

### 8.3 HLTV 交互

- 用户主动点击后才获取最近比赛
- 获取中显示明确加载状态
- 下载中对对应比赛项显示进行中状态
- 下载成功后提供进入下一步的清晰反馈

## 9. 视觉方向

视觉目标不是“营销官网”，而是“现代化桌面应用工作台”。

要求：

- 左栏与右侧内容区层级明确
- 内容面板不再像孤立弹窗盒子
- 版心、留白、字号体系比当前更稳定
- 现代感主要来自结构、留白、层次和状态反馈，而不是花哨装饰

设计约束：

- 保留项目现有深色基调
- 不引入重度装饰性 hero
- 不为了首页视觉牺牲内容效率

## 10. 自适应策略

桌面宽屏下：

- 左栏固定宽度
- 右侧内容区最大化利用宽度

较窄窗口下：

- 左栏允许折叠
- 内容区优先保持单栏可读
- 避免整页级滚动混乱

回放页不受本设计直接修改，但首页壳层的自适应不能干扰现有回放优先级。

## 11. 错误处理

### Demo 库页

- 库为空时展示空状态
- 刷新失败时在页内或状态区给出明确提示

### HLTV 页

- 获取失败时显示失败原因
- 没有可下载比赛时明确说明
- 下载失败时将失败限定在对应比赛项，不污染整个页面

### 首页壳层

- 页面级错误不应让左栏和全局工具栏一起失效

## 12. 验证标准

设计进入实现后，至少应满足以下结果：

1. 打开首页能看到完整 app shell，而不是单卡片页面
2. 左侧导航可在 `Demo 库` 与 `HLTV` 之间切换
3. 默认进入 `Demo 库`
4. `Demo 库` 页仍能完成现有本地 demo 浏览与进入回放
5. `HLTV` 页能承载最近比赛列表与 demo 下载动作
6. 回放页仍然保持独立，不因首页改造被混入首页壳层

## 13. 当前决策

- 首页不做独立仪表盘
- 默认进入 `Demo 库`
- 左栏首批只放 `Demo 库` 与 `HLTV`
- `HLTV` 页第一版只做最近比赛与 demo 下载
- 回放页继续独立，不并入首页标签体系
